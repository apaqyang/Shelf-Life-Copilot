# 贡献与内容边界

本仓库是 **Open Core 开源版**，面向两类读者：

1. **食品厂 IT / 自托管用户** —— 评估、试用、部署产品
2. **开发者** —— 读懂 / 审计 / 贡献代码

## 一条判据：帮人「用产品 / 读代码」才进这里

提交任何**文档**前，问一句：

> 这份文件是帮人**用产品 / 读代码**，还是帮我**卖产品**？

- 帮人用产品 / 读代码 → ✅ 放这里
- 帮我卖产品（定价策略、ROI 话术、销售口径、竞争定位、渠道打法）→ ❌ **不进本仓库**，放私有仓库或发到营销渠道（公众号 / 官网），且发布前删掉一切内部口径

## 该 / 不该进本仓库

| | 判据 | 例子 |
|---|---|---|
| ✅ **技术 / 开发者文档** | 帮人读懂、审计、构建、贡献代码 | `ARCHITECTURE.md`、`TECH_SPEC.md`、`plugins/README.md` |
| ✅ **用户 / 评估文档** | 帮食品厂 IT 评估、试用、部署 | `README.md`、`docs/QUICKSTART.md`、`docs/demo_samples/` |
| ❌ **市场 / 销售内容** | 含定价策略、ROI 话术、销售口径、竞争定位 | 销售脚本、投放 playbook、含"销售口径 X×"的文章 |
| ❌ **对内流程** | 销售 / 演示 / 营销 SOP、内部 TODO、客户商务数据 | demo runbook、年损分档话术、客户线索 |

> **价格 ≠ 销售口径**：在 README 里**公开定价**（如年费分档）是一种刻意的透明，OK；
> 但"销售嘴上说 4-6×（按全采纳假设）"这类**内部口径**绝不能出现在任何公开文件里。

## 红线：这些东西不进公开仓库

- 销售话术 / 报价口径 / "净赚 N 倍"的内部测算假设
- 内容营销 SOP、各平台投放脚本
- 现场 demo 脚本、FAQ 反驳话术、演前 runbook
- 客户名单、线索、商务数据（`data/leads/` 已 gitignore）
- 企业版闭源插件代码（ERP 对接、企微 AES 回调等 —— 见 `plugins/README.md`）

它们分别归档在**私有仓库**里（GTM/销售/营销文档一处、企业版插件代码一处），不在本仓库。

## 提交文档前自查

1. `git diff --cached` 过一遍新增/改动的 `.md`
2. 搜一下敏感词：`grep -rniE "销售口径|净赚|报价|话术|投放|playbook" <你改的文件>`
3. 确认没有指向已下架文件的死链
4. 代码改动：`make check`（ruff + mypy + pytest，等效 CI）必须全绿

## 代码贡献

- 严格 TDD，先写失败测试再实现（见 `.claude/rules/engineering.mdc`）
- 保持 100% 覆盖率；`make check` 全绿才提交
- 改动小而聚焦，不动无关文件
