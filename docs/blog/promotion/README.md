## 🗓️ 推广执行节奏建议

按"成本递增、风险递增"顺序发布。先发不可逆程度低的、可控的，最后发杂志投稿。

### 节奏原则

1. **不堆同一周**：D+0 ~ D+7 内分散，避免被同一批人在多个平台看到引起反感
2. **GitHub 数据先攒**：V2EX 帖子建议等 star ≥ 30 再发，0 star 的开源项目转化率极差
3. **杂志最后**：杂志投稿到出刊 1-3 个月，作为"长尾内容资产"
4. **每发完一站，等 24 小时再发下一站**：观察反馈、修文案、防被举报
